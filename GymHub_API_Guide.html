GymHub API and User Guide

Business and technical onboarding documentation for a new developer: what GymHub does, how the app flow works, how to run it, how to register, how to test, and examples for every API.

1. Product Overview

GymHub is a Spring Boot REST backend for operating gyms. It supports gym owners, employees, and customers using one unified user account. A gym can define services, create subscription packages, register members, sell subscriptions, take payments, activate subscriptions, record attendance, consume invitations, sell extra services, freeze active subscriptions, and manage family package sub-users.

The repository is backend-only. It exposes JSON APIs, Swagger UI, JWT authentication, MySQL persistence, Docker support, and domain services for dashboard and customer mobile app flows.

3. Roles, Permissions, and Security

All protected requests use the JWT access token returned from register/login:

Authorization: Bearer {{accessToken}}
RoleMeaning
SERVICE_PROVIDERGym owner. Creating a gym makes the current user its owner.
EMPLOYEEStaff account. Dashboard actions require an active employee record in the gym.
CUSTOMERGym member/customer. Uses customer self-service APIs.

Employee permissions are: SELL_SUBSCRIPTION, ACTIVATE_SUBSCRIPTION, REGISTER_ATTENDANCE, REGISTER_INVITATION, SELL_EXTRA_SERVICE, MANAGE_SERVICES, MANAGE_PACKAGES, MANAGE_CUSTOMERS, MANAGE_EMPLOYEES, FREEZE_SUBSCRIPTION, MANUAL_ADJUSTMENT, CLOSE_CASH_DAY, ADMIN. The owner bypasses employee permission checks. ADMIN satisfies all employee permissions.

4. Application Flow

  1. Register a user as SERVICE_PROVIDER.
  2. Login and store accessToken.
  3. Create a gym with POST /api/v1/gyms.
  4. Configure settings: partial payment, activation policy, and entrance methods.
  5. Create employees and assign permissions.
  6. Create services, then packages that include services.
  7. Create customers or approve customer link requests.
  8. Sell a subscription and record the initial payment.
  9. If fully paid and activation policy is IMMEDIATE, it activates automatically. If policy is MANUAL, activate it manually.
  10. Record member attendance using member code, customer ID, and optionally subscription ID.
  11. Use invitations for guests, sell extra services, record included-service usage, freeze subscriptions, and manage family sub-users as needed.

6. How to Run

With Docker

  1. Copy .env.example to .env.
  2. Keep fake mail/Cloudinary/Infobip values for normal API testing.
  3. Run the app and MySQL:
docker compose up --build

Default app URL: http://localhost:8085.

Useful URLs

Health:      http://localhost:8085/api/v1/health
Swagger UI:  http://localhost:8085/swagger-ui.html
OpenAPI:     http://localhost:8085/v3/api-docs

Tests

mvn test

Existing tests include the application context and access-control service behavior.

1. نظرة عامة على المنتج

GymHub هو Backend لإدارة الجيمات. النظام يسمح لصاحب الجيم بإنشاء جيم، إضافة موظفين بصلاحيات، إنشاء خدمات وباقات، تسجيل عملاء، بيع اشتراكات، تسجيل المدفوعات، تفعيل الاشتراك، تسجيل الحضور، استخدام الدعوات، بيع خدمات إضافية، تجميد الاشتراك، وإدارة الباقات العائلية.

المشروع الحالي هو REST API فقط باستخدام Spring Boot و MySQL و JWT. لا يوجد Frontend داخل هذا الريبو، لكن يمكن اختبار كل شيء من Swagger أو Postman.

3. الأدوار والصلاحيات

كل API محمي يحتاج Header:

Authorization: Bearer {{accessToken}}
الدورالمعنى
SERVICE_PROVIDERصاحب الجيم. عند إنشاء الجيم يصبح هذا المستخدم هو المالك.
EMPLOYEEموظف داخل جيم. يجب أن يكون Active ولديه الصلاحية المطلوبة.
CUSTOMERعميل أو عضو يستخدم تطبيق العميل.

الصلاحيات المتاحة للموظفين: SELL_SUBSCRIPTION, ACTIVATE_SUBSCRIPTION, REGISTER_ATTENDANCE, REGISTER_INVITATION, SELL_EXTRA_SERVICE, MANAGE_SERVICES, MANAGE_PACKAGES, MANAGE_CUSTOMERS, MANAGE_EMPLOYEES, FREEZE_SUBSCRIPTION, MANUAL_ADJUSTMENT, CLOSE_CASH_DAY, ADMIN. المالك يتجاوز فحص صلاحيات الموظف، وصلاحية ADMIN تعطي كل الصلاحيات.

4. تدفق التطبيق

  1. تسجيل مستخدم بدور SERVICE_PROVIDER.
  2. تسجيل الدخول وحفظ accessToken.
  3. إنشاء جيم من POST /api/v1/gyms.
  4. ضبط إعدادات الدفع والتفعيل وطرق الدخول.
  5. إضافة موظفين وصلاحيات.
  6. إنشاء خدمات ثم إنشاء باقات تعتمد على هذه الخدمات.
  7. إنشاء عملاء أو قبول طلب ربط عميل من تطبيق العميل.
  8. بيع اشتراك وتسجيل أول دفعة.
  9. إذا تم دفع كامل المبلغ وسياسة التفعيل IMMEDIATE يتم التفعيل تلقائيا. إذا كانت MANUAL يتم التفعيل من API التفعيل.
  10. تسجيل حضور العضو باستخدام كود العضوية أو رقم العميل.
  11. استخدام الدعوات، بيع خدمات إضافية، تسجيل استخدام خدمة داخل الباقة، تجميد الاشتراك، وإضافة أعضاء فرعيين للباقات العائلية.
