🏋️ GymHub API User Guide

Complete documentation for Spring Boot backend API

1. What is GymHub?

GymHub is a Spring Boot backend for a complete gym management system. It supports:

  • User registration and login
  • Gym creation and settings management
  • Customers, subscriptions, and payments
  • Employee management
  • Services and subscription packages
  • Attendance tracking
  • Guest invitations
  • Extra service sales

Tech Stack: Spring Boot 3.5.4 · MySQL · Docker · JWT · Cloudinary · Infobip

2. Run the App

Default Startup

  • The app reads SERVER_PORT from environment, defaults to 8085
  • Default base URL: http://localhost:8085

Running Locally with Docker

Terminal
docker compose up --build

Access Swagger UI

Interactive API documentation:

URL
http://localhost:8085/swagger-ui.html

3. Base API Path

All API endpoints start with:

Base URL
/api/v1

Example auth URLs:

POST /api/v1/auth/register
POST /api/v1/auth/login

4. Authentication Flow

4.1 Register a New User

Endpoint:

POST /api/v1/auth/register

Request Body:

JSON
{ "firstName": "Amr", "lastName": "Ali", "email": "amr@example.com", "phone": "01012345678", "password": "Secret123", "role": "SERVICE_PROVIDER" }

Allowed roles:

  • SERVICE_PROVIDER - Gym owner/admin
  • EMPLOYEE - Staff member
  • CUSTOMER - Member/customer

Response:

JSON
{ "accessToken": "eyJ0eXAiOiJKV1QiLCJhbGc...", "refreshToken": "eyJ0eXAiOiJKV1QiLCJhbGc...", "tokenType": "Bearer", "userId": 1, "email": "amr@example.com", "fullName": "Amr Ali", "roles": ["SERVICE_PROVIDER"], "activeContext": "SERVICE_PROVIDER" }

4.2 Login

Endpoint:

POST /api/v1/auth/login

Request Body:

JSON
{ "email": "amr@example.com", "password": "Secret123" }

Response:

Same as register response, containing accessToken and refreshToken.

4.3 Use JWT for Protected Requests

For all non-auth endpoints, include the JWT token in the header:

HTTP Header
Authorization: Bearer

Example cURL:

Terminal
curl -H "Authorization: Bearer eyJ..." \ http://localhost:8085/api/v1/gyms/my

4.4 Get Current Profile

GET /api/v1/auth/me

4.5 Add Another Role

One account can hold multiple roles (e.g., gym owner + customer):

POST /api/v1/auth/add-role?role=CUSTOMER

6. Practical Example: Register, Login & Use

Step 1: Register

cURL
curl -X POST http://localhost:8085/api/v1/auth/register \ -H "Content-Type: application/json" \ -d '{ "firstName":"Amr", "lastName":"Ali", "email":"amr@example.com", "phone":"01012345678", "password":"Secret123", "role":"SERVICE_PROVIDER" }'

Step 2: Login

cURL
curl -X POST http://localhost:8085/api/v1/auth/login \ -H "Content-Type: application/json" \ -d '{ "email":"amr@example.com", "password":"Secret123" }'

Step 3: Use Token

cURL
curl http://localhost:8085/api/v1/gyms/my \ -H "Authorization: Bearer "

7. Quick Tips

💡 Swagger UI: Explore all endpoints interactively at http://localhost:8085/swagger-ui.html
🔓 Public Endpoints: Only /api/v1/auth/register and /api/v1/auth/login are public
🔐 Authentication: All other endpoints require Authorization: Bearer <accessToken>
👤 Multi-Role: Add additional roles to one account via POST /api/v1/auth/add-role?role=CUSTOMER
📄 JWT Duration: Access tokens expire in 24 hours; refresh tokens expire in 7 days

الوثائق الكاملة لـ API خادم Spring Boot

1. ما هو GymHub؟

GymHub هو خادم Spring Boot لنظام إدارة صالة رياضية متكامل. يدعم:

  • تسجيل الدخول والتسجيل
  • إنشاء وإدارة الصالات الرياضية
  • العملاء والاشتراكات والدفع
  • إدارة الموظفين
  • الخدمات وحزم الاشتراك
  • تتبع الحضور
  • دعوات الضيوف
  • بيع الخدمات الإضافية

مكومات التكنولوجيا: Spring Boot 3.5.4 · MySQL · Docker · JWT · Cloudinary · Infobip

4. تدفق المصادقة

4.1 تسجيل مستخدم جديد

نقطة النهاية:

JSON

الأدوار المسموحة:

  • SERVICE_PROVIDER - مالك/مدير الصالة الرياضية
  • EMPLOYEE - موظف
  • CUSTOMER - عضو/عميل

الرد:

JSON
{ "accessToken": "eyJ0eXAiOiJKV1QiLCJhbGc...", "refreshToken": "eyJ0eXAiOiJKV1QiLCJhbGc...", "tokenType": "Bearer", "userId": 1, "email": "amr@example.com", "fullName": "عمرو علي", "roles": ["SERVICE_PROVIDER"], "activeContext": "SERVICE_PROVIDER" }

نقطة النهاية:

POST /api/v1/auth/login

جسم الطلب:

JSON
{ "email": "amr@example.com", "password": "Secret123" }

4.3 استخدام JWT للطلبات المحمية

لجميع نقاط النهاية غير المصادقة، قم بتضمين رمز JWT في رأس الطلب:

رأس HTTP
Authorization: Bearer

مثال cURL:

Terminal

4.5 إضافة دور آخر

يمكن لحساب واحد أن يحتفظ بأدوار متعددة (على سبيل المثال، مالك صالة رياضية + عميل):

POST /api/v1/auth/add-role?role=CUSTOMER
